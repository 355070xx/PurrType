# Credits

PurrType combines original macOS input method code, UI, packaging, privacy controls, and learning/ranking behavior with carefully selected open-source and open-data language resources.

This page records those resources so their authors and licenses stay visible. PurrType's MIT License applies to this project's original source code; bundled dictionary, table, and data resources keep their own upstream license terms.

## Rime Cangjie

PurrType includes selected Cangjie dictionary data from the Rime Cangjie project.

- Source: https://github.com/rime/rime-cangjie
- Local path: `third_party/rime-cangjie`
- Included source files: `cangjie5.dict.yaml`, `cangjie5.base.dict.yaml`, `cangjie5.extended.dict.yaml`, `cangjie5_express.schema.yaml`, `AUTHORS`, `LICENSE`
- Runtime package materials: `cangjie5.base.dict.yaml`, `cangjie5.extended.dict.yaml`, `AUTHORS`, `LICENSE`

The dictionary headers and upstream license files are kept with the repository and release package. Treat this as upstream copyleft dictionary data; do not describe a full binary release payload as MIT-only.

## Rime Luna Pinyin

PurrType includes Luna Pinyin dictionary data as part of its Traditional Chinese Pinyin candidate support.

- Source: https://github.com/rime/rime-luna-pinyin
- Local path: `third_party/rime-pinyin`
- Included files: `luna_pinyin.dict.yaml`, `AUTHORS`, `LICENSE`

The upstream license and authors files are kept with release materials that include this dictionary data.

## IBus Table Chinese

PurrType uses selected open table data from IBus Table Chinese for Cangjie and classic Sucheng candidate coverage.

- Source: https://github.com/mike-fabian/ibus-table-chinese
- Debian source package: `ibus-table-chinese_1.8.14.orig.tar.gz`
- Local path: `third_party/ibus-table-chinese`
- Included files: `cangjie5.txt`, `quick-classic.txt`, `LICENSE`, `README.md`

The imported table headers state that the tables are freely redistributable without restriction. PurrType keeps those headers and attribution files with release materials that include these tables.

## McBopomofo Associated Phrases

PurrType uses McBopomofo associated phrase data as one seed source for related-candidate generation.

- Source: https://github.com/openvanilla/fcitx5-mcbopomofo
- Local path: `third_party/mcbopomofo`
- Included files: `associated-phrases-v2.txt`, `LICENSE.txt`, `README.md`
- License: MIT

PurrType does not use McBopomofo's Bopomofo input-code data as an input table. Runtime packages include the McBopomofo MIT license because generated association data derives in part from this phrase resource.

## Hong Kong Supplementary Character Set

PurrType includes official Hong Kong Supplementary Character Set data to improve Hong Kong Traditional Chinese character coverage.

- Source: https://data.gov.hk/en-data/dataset/hk-dpo-dpo_hp-hong-kong-supplementary-character-set-related-information
- JSON: https://www.digitalpolicy.gov.hk/open_data/ccli/HKSCS2016.json
- Local path: `third_party/hkscs`
- Included files: `HKSCS2016.json`, `README.md`, `TERMS.md`

The local terms reference is kept with release materials that include this data.

## Chinese Open Desktop CIN Tables

PurrType keeps one Chinese Open Desktop CIN table in the source tree for audit and legacy comparison.

- Source: https://github.com/chinese-opendesktop/cin-tables
- Local path: `third_party/cin-tables`
- Included files: `mscj3.cin`, `LICENSE`, `README`
- License: CC0-1.0

This table is not used as a runtime candidate table.

## Generated Association Data

`resources/association_generated.tsv` is generated by PurrType tooling from project-local seed TSVs, typing corpora, and the acknowledged upstream language resources above. Regenerate it with:

```sh
make update-association-seeds
```
